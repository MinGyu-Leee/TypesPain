# 불경 문답 — 개발 로드맵

## 확정된 포지션

> **"2500년 된 분류 체계로, 지금 내 고통에 이름을 붙이세요."**

**앱 정체**: 불교 심리학 × 정서 진단기  
**핵심 기제**: Affect Labeling — 감정에 이름 붙이는 행위 자체가 편도체 반응을 줄이는 신경과학적 효과  
**기술 방향**: ML 미사용. 키워드+패턴의 일관성이 앱 정체성. (Vercel 배포 제약으로도 ML 불가)

---

## 현재 작업 (배포 전)

### 1. 키워드 사전 확장 — 진행 중

팔고 분류 정확도를 최대한 끌어올리는 단계.
테스트 케이스 30개 기준으로 검증 후 배포 판단.

목표: 다양한 표현 방식의 한국어 단문에서 안정적인 분류.

### 2. 배포

키워드 안정화 후 Vercel 배포.

---

## 향후 검토 (배포 후 사용자 데이터 기반)

배포 이후 실제 사용자 입력 패턴을 보고 나서 결정.

### 고통의 지도

사용자의 팔고 패턴을 시간 흐름으로 기록·시각화.
- 저장소: localStorage (로그인 불필요)
- 시각화: 최근 30일 팔고 빈도 bar chart
- 인사이트: "이번 달 당신은 주로 구부득고 상태였습니다"

### OG 이미지 공유

"나는 구부득고(求不得苦)입니다" 공유 이미지 생성.
- `/api/og` 엔드포인트로 팔고 카드 이미지 동적 생성

### 오온성고 UX 개선

진짜 오온성고(존재론적 불안, 자기혐오)와 분류 실패 fallback을 사용자가 구분하는 경험 설계.
- "조금 더 구체적으로 말해줄 수 있나요?" 재질문 UX 고려

### 팔고 상세 설명 예시

각 팔고별로 "이런 상황에서 느끼는 고통이에요" 예시 문장 1~2개 추가.
